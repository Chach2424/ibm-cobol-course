Data Division

All of the data we use in our COBOL program gets set up in the DATA DIVISION, and we use a data relationship to add structure to it.

MOVE and COMPUTE

These are two reserved words we use to manipulate the values of variables. MOVE moves data from one place into a variable. COMPUTE is used to do mathematical operations.

File Handling

This would involve the SELECT and ASSIGN clauses in the ENVIRONMENT DIVISION, FD statements in the DATA DIVISION, and in the PROCEDURE DIVISION, we'd be using the OPEN, CLOSE, READ-INTO and WRITE-FROM statements

The ENVIRONMENT and DATA divisions are where inputs and outputs are described, which in turn are used in the PROCEDURE DIVISION

Since we're working with z/os, the datasets we'll be using will be of the sequential data storage type. A dataset in z/os consists of records. Each record is a line in a dataset, and has a defined length. In COBOL, data can be loaded as a block for efficiency, where a block is a group of records, and put in a buffer.
