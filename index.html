Introduction

COBOL is an acronym for Common Business-Oriented Language. It was designed for data-processing with accuracy and efficiency in mind, as well as for ease in reading and writing. It was invented in 1959 by a group of programmers who wanted a programming language that could run on the computers made by different manufacturers. It was partially based on a programming language called FLOW-MATIC that was developed by Grace Hopper.

COBOL's ability to process large amounts of data accurately and efficiently has led to its continued use into the present day. Banks still use it, and in any transaction made with a credit or debit card, there's a high chance that COBOL was involved in the background.

COBOL written for the IBM mainframe operating system called z/os is described as "enterprise COBOL". The COBOL code you see in the IBM course "Learning COBOL Programming with VSCODE" is enterprise COBOL. It's compiled and executed differently than COBOL written for other platforms. But it still follows COBOL standardization.

Basic COBOL Syntax

Columns

COBOL is column-dependent. It looks for certain things to be in certain positions. It looks in 5 primary areas of a 72-character line:

  1. Sequence Number Area: columns 1-6

    Sometimes blank, sometimes used to provide context to a series of statements

  2. Indicator Area: column 7

    A multi-purpose area. An asterisk denotes a comment. A dash or hyphen denotes the continuation of a previous line.A "D" is used for a debugging line. And a slash is used for source code listing.

  3. A Area: columns 8-11

    Here you'll find elements that give COBOL programs structure, e.g. divisions, sections, paragraphs, etc.

  4. B Area: columns 12-72

    This is where statements, sentences and clauses go, i.e. the things that make a COBOL program "do things".

  5. Identification Area: 73-80

    The compiler ignores this part. It's an area that the programmer can use for any purpose. It's often left blank.

Reserved Words

There are also certain words in COBOL that are reserved, meaning that they serve a specific purpose programmatically and you should be careful not to use them. Examples include:

  • READY
  • UNTIL
  • SET
  • IF
  • DATA
  • FILE
  • END

COBOL Divisions

COBOL has a hierarchical structure:

  1. DIVISIONS

    There are 4 COBOL DIVISIONS:

    1. Identification DIVISION

      Where you put the name of your program, who the programmer was, when it was written and how it should be used.

    2. Environment DIVISION

      This could mean two things: how you set the kind of computer environment that runs the program, hwo you set the mapping between the files in your program and the files in the dataset.

    3. Data DIVISION

      Sets up all the data that would be used in your program, e.g. files, data from other programs, what kind of storage or data would be used while the program is run, and what the ouput would be when the program has finished running.

    4. Procedure DIVISION

      This is where SECTIONS and PARAGRAPHS go. The code that "does things" goes in the procedure DIVISION.

  2. SECTIONS

    PARAGRAPHS go into SECTIONS (duh). There are all sots of them and their names can be user-define or pre-defined. They can be used to structure your program, or to make sure certain information is where it needs to be.

  3. PARAGRAPHS

    PARAGRAPHS contain SENTENCES (also duh). PARAGRAPHS are defined by the user to give their program structure. PARAGRAPHS generally represent a specific action made up of smaller steps. If the action is something that might be called several times in your program, it would go inside a paragraph. You would then call that PARAGRAPH elsewhere in the program

  4. SENTENCES

    SENTENCES are made up of STATEMENTS (duh again). A COBOL SENTENCE can be made up of 1 or more STATEMENTS, serving as a logical container. A period at the end of a SENTENCE is an implicit scope terminator, and declares as finished all previous STATEMENTS that have not yet ended.

    Alternatively, you can also end a SENTENCE or STATEMENT with an explicit scope terminator, e.g. ending an IF STATEMENT with an END-IF.

  5. STATEMENTS

    A single directive that usually begins with one of the reserved words previously mentioned.

COBOL Variables

It's common for a programming language to have variables and that is also the case with COBOL. A variable is a name defined by the programmer to represent a value. It's referred to as a dataname, and can be anything as long as it:

  • has 30 characters or less
  • doesn't contain any spaces
  • is composed of only letters, numbers or hashes
  • isn't a reserved word

When we declare variables in COBOL, we have to be explicit in what type of variable it's going to be:

  • numeric
  • alphabetic
  • both

Numeric Variables

We also specify how many of those numbers and/or letters we'll be using. We set the length and data type do in the picture clause, or pic. For example, PIC 9 would be a single numeric value where the length is one. If we need more than one length, we specify it with a parenthesis. For example, PIC 9(5) would be a numeric value with 4 digits, which would be an appropriate variable to store a year.

Alphabetic Variables

Similarly, PIC A would be an alphabetic variable with one character. PIC X(8) would be an alphabetic variable with 8 characters, which would be appropriate for a car license plate number

There are many other types of PIC clauses. For now let's just focus on the three we've covered. We use picture clauses to represent symbols that appear in the value. For example, PIC 9(4)V99 would be a numeric variable with 4 digits, then a decimal point, and then two more digits. That could perhaps be a frequency like 2458.75GhZ. Something like PIC $9,999V99 is obviously more suitable for sums of money.

We usually think of variables as changing values at some point. COBOL has something called literals which contain data that will not change as the program runs. We might use it for unchanging values that we use several times in a program, and would only have to set it once. COBOL has some figurative literals built in. For example:

  • ZERO / ZEROES
  • SPACE / SPACES
  • LOW-VALUE / HIGH-VALUE
  • NULL / NULLS
